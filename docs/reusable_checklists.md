# Reusable Checklists

## Video Brief Checklist

- [ ] Who is the target viewer?
- [ ] What business process is being improved?
- [ ] What is painful about the current process?
- [ ] What does the product automate or assist?
- [ ] What remains human-reviewed?
- [ ] What are the 3-4 most important features?
- [ ] What KPI language should appear in the close?
- [ ] What should not be shown publicly?

## Storyboard Checklist

- [ ] Opening line is clear
- [ ] Current burden is concrete
- [ ] Product workflow has simple steps
- [ ] Product screenshots appear at the right moments
- [ ] KPI scene is specific
- [ ] Closing line names the product category

## Subtitle Checklist

- [ ] Captions match voiceover
- [ ] Captions do not overflow
- [ ] Captions do not cover key UI
- [ ] No confidential names appear
- [ ] Scene changes align with caption meaning

## HyperFrames QA Checklist

- [ ] Lint passes
- [ ] Render completes
- [ ] Audio duration matches video duration
- [ ] Screenshots are readable
- [ ] Focus boxes align with UI elements
- [ ] No unwanted jump cuts
- [ ] No off-brand colors
- [ ] No logo or confidential data visible

## GitHub Publishing Checklist

- [ ] README is clear
- [ ] Case study is anonymized
- [ ] Prompts are reusable
- [ ] No raw internal assets included
- [ ] No API keys or credentials
- [ ] No real company names
- [ ] Short snippets are sanitized
- [ ] License included
